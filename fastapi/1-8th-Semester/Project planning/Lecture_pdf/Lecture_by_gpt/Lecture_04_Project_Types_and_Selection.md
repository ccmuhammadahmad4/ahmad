## Introduction

Projects differ greatly in:
- **Sector**: IT, construction, healthcare, R&D, social

## Why Project Selection is Important

Organizations have limited resources but unlimited ideas. Cannot do everything!

### Reasons for Project Selection:

#### 1. **Limited Resources**
- Money is finite
- People are finite
- Time is finite
- Must choose wisely

#### 2. **Strategic Alignment**
- Ensure projects support business goals
- Not every good idea fits strategy
- Focus on what matters most

#### 3. **Risk Reduction**
- Some projects are too risky
- Better to reject than fail
- Protect organization's resources

#### 4. **Opportunity Cost**
- Doing Project A means NOT doing Project B
- Choose the project with highest value
- Maximize return on investment

#### 5. **Improve Success Rate**
- Select projects you CAN succeed at
- Don't overcommit
- Build on strengths

### Without Proper Selection:

```
âŒ Wasted resources
âŒ Failed projects
âŒ Missed opportunities
```

### With Proper Selection:

### Category B: Numeric Models (Quantitative)

Use mathematical calculations and financial analysis.

#### 1. **Payback Period (PP)**

**Definition**: 
Time required to recover initial investment.

**Formula**:
```
Payback Period = Initial Investment / Annual Cash Inflow
```

**Example**:
```
Investment: PKR 1,000,000
Annual Return: PKR 250,000
Payback Period = 1,000,000 / 250,000 = 4 years
```

**Decision Rule**:
- Shorter payback = Better
- Choose project with fastest return

**Pros**:
- âœ… Simple to calculate
- âœ… Easy to understand
- âœ… Focus on liquidity

**Cons**:
- âŒ Ignores time value of money
- âŒ Ignores returns after payback
- âŒ Ignores project risk

**Best for**: 
- Cash-constrained organizations
- High-risk industries
- Short-term focus

---

#### 2. **Net Present Value (NPV)**

**Definition**: 
Today's value of all future cash flows minus initial investment.

**Concept**:
Money today is worth more than same money tomorrow (time value of money).

**Formula**:
```
NPV = Î£ [Cash Flow / (1 + r)^t] - Initial Investment

Where:
r = Discount rate (interest rate)
t = Time period
```

**Example**:
```
Project costs: PKR 500,000
Year 1 return: PKR 200,000
Year 2 return: PKR 200,000
Year 3 return: PKR 200,000
Discount rate: 10%

NPV = [200,000/(1.1)^1] + [200,000/(1.1)^2] + [200,000/(1.1)^3] - 500,000
NPV = 181,818 + 165,289 + 150,263 - 500,000
NPV = PKR 497,370 - 500,000
NPV = -PKR 2,630 (NEGATIVE - Don't invest!)
```

**Decision Rule**:
- NPV > 0 â†’ ACCEPT (creates value)
- NPV < 0 â†’ REJECT (destroys value)
- NPV = 0 â†’ Indifferent

**Comparison**:
- Project A: NPV = +500,000
- Project B: NPV = +300,000
- **Choose A** (higher NPV)

**Pros**:
- âœ… Considers time value of money
- âœ… Considers all cash flows
- âœ… Shows actual value creation
- âœ… Most theoretically sound

**Cons**:
- âŒ Complex calculation
- âŒ Requires accurate forecasts
- âŒ Choosing discount rate is subjective

**Best for**: 
- Long-term projects
- Capital-intensive projects
- Mature organizations

---

#### 3. **Internal Rate of Return (IRR)**

**Definition**: 
Discount rate at which NPV equals zero. The project's "rate of return."

**Concept**:
At what interest rate does the project break even?

**Decision Rule**:
- IRR > Required return â†’ ACCEPT
- IRR < Required return â†’ REJECT
- Higher IRR = Better project

**Example**:
```
Project A: IRR = 15%
Project B: IRR = 12%
Company's required return: 10%

Both acceptable, but A is better.
```

**Pros**:
- âœ… Easy to understand (percentage)
- âœ… Compares well across projects
- âœ… No need to know discount rate

**Cons**:
- âŒ Complex calculation
- âŒ May give misleading results with unconventional cash flows
- âŒ Doesn't show absolute value

**Note**: 
> "If IRR and NPV conflict, trust NPV" - Finance experts

**Best for**:
- Comparing similar-sized projects
- When discount rate is uncertain
- Communicating with non-financial stakeholders

### Category C: Scoring/Ranking Models (Multi-Criteria)

Evaluates projects on multiple dimensions, not just financial.

**How it works**:
1. Identify criteria (cost, benefit, risk, strategic fit, etc.)
2. Assign weights to each criterion (based on importance)
3. Score each project on each criterion (1-5 scale)
4. Calculate weighted total score
5. Choose project with highest score

**Cons**:

### Choosing the Right Method

```
IF urgent crisis
    THEN use Urgency method

ELSE IF legally required
    THEN use Operating Necessity

ELSE IF primarily financial decision
    THEN use NPV or IRR

ELSE IF multiple competing factors
    THEN use Scoring Model

ELSE IF CEO insists (politics)
    THEN Sacred Cow (no choice!)
```

### Project Type Matters

Different project types may use different methods:

- **Personal**: Intuitive judgment
- **Local**: Scoring model (community benefit, cost, feasibility)
- **Organizational**: NPV, IRR, Scoring
- **National**: Cost-benefit analysis, political factors, public benefit
- **Global**: Complex multi-criteria, international consensus
